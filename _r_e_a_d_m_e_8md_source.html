1 Kinetic C Client Library
2 ========================
3 ![](http://travis-ci.org/atomicobject/kinetic-c.png?branch=master)
4 
5 This repo contains code for producing Kinetic C clients which use the Seagate Kinetic protocol. Code examples are included for reference as part of a utility (`kinetic-c-client-utility`) that builds and links against an installed Kinetic C static library.
6 
7 The project Makefile can be used as a reference for developing a Makefile for building for a new custom Kinetic C client.
8 
9 [Kinetic-C build status](http://travis-ci.org/atomicobject/kinetic-c) is provided via [Travis CI](http://travis-ci.org)
10 
11 Prerequisites
12 -------------
13 * [Open SSL](https://www.openssl.org/) for security and encryption
14  * Installation (if you don't already have OpenSSL installed)
15  * Linux (using apt-get)

17  * OSX (using [Homebrew](http://brew.sh/))

48 Examples
49 ========
50 
51 The following examples are provided for development reference and as utilities to aid development. In order to execute a given example, you must first do:
52 
53  > cd build/artifacts/release
54 
55 You can then execute `kinetic-c-client-util` with a valid example name, optionally preceeded with any of the optional arguments.
56 
57 **Options**
58 * `--host [HostName/IP]` or `-h [HostName/IP]` - Set the Kinetic Device host
59 * `--tls` - Use the TLS port to execute the specified operation(s)
60 
61 **Operations**
62 * `kinetic-c-client [--host|-h hostname|123.253.253.23] noop put get`
63  * `kinetic-c-client-util noop`
64  * Execute a NoOp (ping) operation to verify the Kinetic Device is ready
65  * `kinetic-c-client-util put`
66  * Execute a Put operation to store a key/value
67  * `kinetic-c-client-util get`
68  * Execute a Get operation to retrieve a key/value
