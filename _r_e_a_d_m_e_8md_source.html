1 Kinetic C Client Library
2 ========================
3 
4 This repo contains code for producing Kinetic C clients which use the Seagate Kinetic protocol.
5 
6 Prerequisites
7 -------------
8 * [Open SSL](https://www.openssl.org/) for security and encryption
9  * Installation (if you don't already have OpenSSL installed)
10  * Linux (using apt-get)
11  * `> sudo apt-get install openssl`
12  * OSX (using [Homebrew](http://brew.sh/))
13  * `> brew install openssl`

55 Example Client/Test Utility
56 ===========================
57 
58 Code examples are included for reference as part of a test utility. The source code for the utility is used to build both a static and dynamically linked verion of the kinetic-c-client library.
59 
60 * 'kinetic-c-util' builds/links against installed Kinetic C static library (.a)
61 * 'kinetic-c-util.x.y.z' builds/links against installed Kinetic C dynamic library (.so)
62 
63 The project Makefile can be used as a reference for developing a Makefile for building for a new custom Kinetic C client.
64 
65 In order to execute a given example, after building it, you must first do:
66 
67  > cd bin
68 
69 You can then execute `kinetic-c-client-util` with a valid example name, optionally preceeded with any of the optional arguments.
70 
71 Options
72 -------
73 * `--host [HostName/IP]` or `-h [HostName/IP]` - Set the Kinetic Device host
74 * `--tls` - Use the TLS port to execute the specified operation(s)
75 
76 Operations
77 ----------
78 * `kinetic-c-client [--host|-h hostname|123.253.253.23] noop put get`
79  * `kinetic-c-client-util noop`
80  * Execute a NoOp (ping) operation to verify the Kinetic Device is ready
81  * `kinetic-c-client-util put`
82  * Execute a Put operation to store a key/value entry
83  * `kinetic-c-client-util get`
84  * Execute a Get operation to retrieve a key/value entry
85  * `kinetic-c-client-util delete`
86  * Execute a Delete operation to destroy a key/value entry
